## 3. Data pipeline (`pipeline/build_dataset.py`)

Makes the data **connected + reproducible** (the 10% line) instead of pasted-in.

- **Input:** `Knowledge/data/processed/employment_equity_department_gaps.csv`.
- **Transforms:** cast percents to float; compute `expected_number`, `gap`, `representation_percent` and **assert** they reconcile with stored values (fail loud on mismatch); compute `severity` (Slight <2 / Moderate 2–4.9 / Substantial 5–9.9 / Severe ≥10 pp; `above` if at/over benchmark) **in code, not baked offline**; compute `priority` = bottom-quartile of gap per group/year; attach `prior_gap`/`prior_rep_pct`/`has_trend` for the 35 two-year departments; carry the **3 valid** PSES columns only.
- **Output:** `app/data/equity.json` — a typed, documented array; plus a tiny `meta.json` (row counts, WFA-by-year table, suppression count, generation note).
- **Guard:** the pipeline runs `edi-data-guard`'s `validate.py` on the CSV first; if it exits non-zero, the build stops (the guard becomes load-bearing, mirroring the course model).
- **Docs fix:** update `data_dictionary.md`, `data_profile.md`, `CLAUDE.md`, `what_claude_cannot_guess.md`, `domain_research_note.md` to describe **3** PSES indicators; fix the gold example to the verified RCMP·PwD row; delete the stale root CSV.

## 4. Application architecture (the six views)

Single data source (`equity.json`) loaded once; views are client components that filter/derive. State: a small URL-synced store (group filter + active dept) so views deep-link and the back button works (fixes the old no-routing problem). Every rep% renders with **N and WFA beside it** (hard spec rule) in *every* view including Frame.

1. **Frame** — headline KPIs (depts, groups, # priority flags, avg gap by group), a severity distribution, and an animated "headline number." Sets context + the always-visible responsible-use caveat.
2. **Explore Gaps** — the core table: ranked deficit bars (rep% vs WFA benchmark line), priority ⚑, severity, signal text. Filter by group, sort by gap. This is the decision surface.
3. **Compare** — cross-group severity heatmap; surfaces departments below benchmark for ≥2 groups. Accessible (not color-only: severity also shown as label/pattern).
4. **Track** — year-over-year movement for the 35 two-year departments, with the benchmark-shift caveat (a narrowing gap may be benchmark-driven, not improvement). Only drawn for depts present both years.
5. **Present** — pick a dept×group → the 5-block summary (Key Finding · Evidence · Caveat · Human Review · Next Action), generated by the `presentation-summary-render` logic; plus distribution charts. Exportable.
6. **Ask** — real conversational Q&A (see §6).

**Charts:** ported from the current app's *correct* SVG math (viewBox + width:100% so they never overflow), wrapped as React components with proper titles/labels and `<title>`/`aria-label` text alternatives.

## 5. Design system & aesthetic direction (per frontend-design)

**Concept — "The Public Accountability Ledger."** This is a government equity-accountability instrument, not a SaaS toy. The direction is **editorial / data-journalism**: serious, authoritative, ink-on-paper gravitas with modern precision — the feel of a beautifully typeset public report crossed with a data terminal. Striking through typography, restraint, and motion — explicitly avoiding the purple-gradient-on-white "AI slop" look.

- **Typography (distinctive, not Inter/Roboto):**
  - Display/headlines: **Fraunces** (variable high-contrast serif) — editorial authority + character.
  - Body/UI: **Geist** (clean, modern, fitting for a Vercel deploy; not generic).
  - Data/figures: **Geist Mono** with tabular numerals for tables and stat readouts — gives the "instrument" feel and aligns columns.
- **Color — documentary palette (light, with a dark-mode option):** warm paper `#F6F3EC`, deep ink `#16140F`, with a single, carefully-used **signal accent** for "below benchmark" (a considered amber→red ramp for severity) and a calm slate/teal for "at/above." Color is **never the sole carrier** of status (a11y) — severity is always also a label, and the heatmap uses text + pattern.
- **Signature elements:** (a) the **deficit bar** — a department's representation drawn as a bar that stops short of an inked benchmark line, with the gap rendered as a measured void; (b) a **Frame** landing with a large Fraunces headline and an animated count-up to the headline number; (c) a quiet "session pipeline / governance" rail showing active guardrails (kept, but truthful).
- **Motion (subtle, high-impact, per frontend-design + `rendering-animate-svg-wrapper`):** one orchestrated page-load with staggered reveals; number count-ups; smooth view transitions; hover states on bars revealing the N/WFA detail. Respect `prefers-reduced-motion`. Use the Motion library for React where it earns its place; CSS for the rest.
- **Accessibility (a hard requirement, fixes the worst current defect):** semantic landmarks + heading hierarchy, real tab semantics (`role`/`aria-selected`/keyboard), focus management on view change, text alternatives for every chart, WCAG-AA contrast, full keyboard path, `prefers-reduced-motion`. Target: a clean axe/Lighthouse a11y pass.

> This direction is a recommendation — if you'd prefer a darker "terminal/brutalist" or a lighter "Swiss/Statistics-Canada-modern" take, say so and I'll adjust before building.

## 6. The real "Ask" tab (turns the biggest liability into the strongest demo)

- **Route:** `app/api/ask/route.ts` — server-side, streaming. Holds the API key server-side (never reaches the browser), per `server-auth-actions`.
- **Grounding:** the relevant `equity.json` slice (selected dept and/or comparative rows) is passed as structured context so answers are grounded in the real data, with N/WFA always present.
- **Guardrails in the system prompt** (this is where `question-router` + `interpretation-guardrails` become *real*): refuse staffing/hiring/firing questions; refuse causal/blame framing and reframe to "signal for review"; never average groups into a composite; always cite N + WFA; only use the approved review vocabulary; append the TBS attribution + responsible-use caveat. The same refusals the old regex faked — now actually enforced by the model under instruction, and demonstrably so.
- **UX:** streaming responses, suggested starter questions, graceful "I can't answer that / here's why" for out-of-scope, and an **action-plan** generator (the 5-block format) as a structured output.
- **Cost/perf:** small context, `claude-sonnet-4-6` default; `bundle-dynamic-import` the chat UI; defer until the tab is opened.
- **Governance:** the route logs each Q→route-decision (answer/reframe/refuse) for the deployment log / eval, and a human-review note is shown for any actionable suggestion.
